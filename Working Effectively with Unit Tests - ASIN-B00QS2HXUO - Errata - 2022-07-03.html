Highlight (yellow) - Page 312
Test Data Builders I find Test Data Builders to be a superior option. The guidelines for creating a Test Data Builder are found below. For each class you want to use in a test, create a Builder for that class that: Has an instance variable for each constructor parameter Initializes its instance variables to commonly used or safe values Has a build method that creates a new object using the values in its instance variables Has “chainable” public methods for overriding the values in its instance variables.–Nat Pryce, Test Data Builders Creating a test fixture can be both tedious and error prone.
Note - Page 312
There's a strong bias for test data-builder so far that it implies a non-existing conflict with object-mother. Another important concern is breaking abstraction by introducing changing internal object state using `w` in production code. But I will assume the builders are only available in test code/builds.
