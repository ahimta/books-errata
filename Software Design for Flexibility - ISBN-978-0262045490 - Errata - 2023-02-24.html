6: Layering
Highlight (yellow) - Page 322
However, sometimes it is appropriate to attach a detailed audit history describing the derivation of a data item, to allow some later process to use the derivation for some purpose or to evaluate the validity of the derivation for debugging.
Note - Page 322
Important to stress the point for development/debugging where the program inputs are relatively small and there's plenty of space/time resources. The same applies for assertions/pre-conditions/post-conditions. Which should be used in production too as a crash/system-graceful-shutdown is better than unexpected state/input/output which can cause much more damage. GHC (the Haskell compiler) is one example of a multi-decade project with many great practices that are not very known but still very effective and intuitive. One of which is expensive assertions in development. I'd expect this book to cover some of these in quite some depth as more evolvable code can be harder to develop. And without proper practices, trying to implement reasonably general implementations can fail and lead to bouncing back to overly specific implementations.
